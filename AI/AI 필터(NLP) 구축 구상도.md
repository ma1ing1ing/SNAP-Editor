# AI 필터(NLP) 구축 구상도

1단계 : 판단 기준점 (품사 정의하기)
지워야 할 타켓 단어들이 문법적으로 어떤 역할을 할 때만 지울 것인지 기준을 세운다.

```
타겟 단어 : "어", "음", "아", "그"
타겟 품사 : 감탄사(Interjection)
    단순히 글자가 같다고 지우는 것이 아닌, AI (Kiwi)가 문장을 분석했을 때 해
    단어에 감탄사(IC) 꼬리표가 붙었을 때만 컷편집의 대상으로 한다.
```

2단계 : 2-2way 크로스 체크 로직 (핵심 엔진)

```
단어 하나가 들어올 때마다 AI가 두 번의 질문을 던져 교차 검증을 하도록 로직을 짠다.
    1. 1차 검문 (사전 매칭)
        해당 단어가 지우기로 한 불용어 리스트에 있는가를 판단한다.
        ex) Yes -> 2차 검문으로 패스 / No -> 정상 단어로 통과
    2. 2차 검문(문맥 파악)
        해당 단어의 품사가 감탄사(IC)로 사용 되었는지 판단한다.
        Yes -> 삭제 확정으로 컷편집 리스트에 추가/ No -> 정상 단어로 통과
```

3단계 : 햇갈리기 쉬운 예외 케이스(Edge Case) 방어
    2-Way 크로스 체크가 도입되면 기존 코드의 치명적 오류들이 자동으로 방어된다.

```
    **상황 A** ("밥 먹었어"): '어'가 불용어 리스트에 있어 1차 검문을 통과하지만
    2차 검문에서 AI가 이를 '종결 어미(EF)'로 판단하므로 삭제되지 않는다.
    (방어성공)

    **상황 B** ("저기 그 차"): '그'가 1차 검문을 통과하지만, 2차 검문에서 AI가
    이를 '관형사(MM)'로 판단하므로 삭제되지 않는다. (방어 성공)

    **상황 C** ("음... 그니까"): 1차 검문 통과 후, '음'을 '감탄사(IC)'로 정확히
    판단하여 삭제 대상으로 확정한다.. (타격 성공)
```

4단게 : 듀얼 출력 (자막 & 컷편집 분리)
    AI 필터를 통과한 결과물은 목적에 따라 두 갈래로 나뉘어 출력되도록 파이프라인을 
    구성한다.

```
    출력 1 : 텍스트 에디터용으로 필터링된 단어만 빼고 다시 자연어스럽게 이어 붙인 자막
    출력 2 : 틸터링에 걸린 단어들이 말해진 정확한 시작 시간(Start)과 끝 시간(End)만 모아둔 데이터 셋
```
